SECTION 6 — FORBIDDEN PATTERNS REGISTRY — FULL CURRENT LIST
================================================================================

FP-001: SentenceTransformer() direct instantiation
  Use AutoTokenizer + AutoModel from transformers instead.

FP-002: urllib.request for external API calls
  Use http.client.HTTPSConnection exclusively.

FP-003: Windows file paths in Lambda code
  Lambda runs Linux. Use /var/task/ paths only.

FP-004: Hardcoded Groq model names without Knowledge Retrieval confirmation
  Models deprecate without notice. Use v3r-keys table — never hardcode.

FP-005: win_amd64 or cp314 wheels in Docker build
  Lambda requires manylinux x86_64 cp312 only.

FP-006: Inline JSON strings in PowerShell AWS CLI
  PowerShell 5.1 mangles quotes. Always use WriteAllText + file:// absolute path.

FP-007: Relative paths in AWS CLI file:// references
  Always use absolute paths.

FP-008: Missing import os when using os.environ
  Causes NameError at Lambda cold start.

FP-009: ECR login skipped when 12+ hours elapsed
  Results in 403 Forbidden on push after build completes.
  NOTE: ECR Public tokens also expire after 12 hours — same rule applies.

FP-010: Missing --provenance false --sbom false from docker buildx build
  Produces OCI attestation manifests that break Lambda image deployment.

FP-011: Omitting --platform linux/amd64 from docker buildx build
  Docker Desktop defaults to host architecture.

FP-012: Assuming Docker pruning frees Windows disk space immediately
  docker_data.vhdx does not auto-shrink. Run diskpart compact vdisk after cleanup.

FP-013: Assuming Groq API key is permanently valid
  Groq can deactivate keys without notice. Keys now in v3r-keys — single update
  point if key changes.

FP-014: Full app.py replacement without visual audit of existing file
  Existing file has incompatible signatures and structure. Surgical edits only
  after photo review.

FP-015: Omitting --cli-binary-format raw-in-base64-out from aws lambda invoke
        with --payload file:// commands
  AWS CLI v2 rejects plain JSON payload files without this flag.

FP-016: Omitting $ from $(LAMBDA_TASK_ROOT) in Dockerfile COPY instructions
  Handler copied to wrong path. Lambda throws Runtime.ImportModuleError.

FP-017: Syncing credentials to Drive via rclone or any other method
  AWS keys, API keys, tokens must never appear in any file pushed to Drive.

FP-018: Using --delete flag with rclone in CONDUIT scripts
  Drive is the archive. Use rclone copy with --no-delete only. Never rclone sync.

SECTION 8 — ACTIVE ISSUE — ECR PUBLIC CREDENTIAL STORE ERROR
================================================================================

Error: "error storing credentials - error exit status 1; out: the stub
received bad data"

Root cause: Docker Desktop credential helper (docker-credential-wincred or
docker-credential-desktop) on Windows 10 failing to store ECR Public auth
token. Known issue on Windows 10 with oversized tokens from AWS ECR Public.

Fix procedure:
  1. Open: C:\Users\Source-1\.docker\config.json in Notepad++
  2. Locate: "credsStore" field
  3. If value is "wincred" or "desktop", change to ""
  4. Save file
  5. Retry: aws ecr-public get-login-password --region us-east-1 | docker
     login --username AWS --password-stdin public.ecr.aws
  6. If Login Succeeded: proceed to buildx build and push
  7. After push completes: restore credsStore to original value if desired

This is the active blocker for ST-01. All other ST-01 steps are ready to
execute immediately after this fix.
