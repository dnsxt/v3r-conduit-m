STATUS: NEW FILE — ~140 lines
LOCATION: C:\Users\Source-1\Desktop\rag-deploy\ecr-build\lambda_router.py

PURPOSE:
  Shared routing utility. Imported by all V3R Lambda handlers via:
    from lambda_router import route_llm
  Exposes single public function: route_llm(). All direct provider calls
  eliminated from individual handlers. Handlers never know which provider
  executed — they receive normalized text response only.

IMPORTS:
  json, boto3, http.client, datetime.date

KEY FUNCTIONS:

  get_route(route_id)
    Fetches routing record from v3r-routing table by route_id.
    Falls back to 'default' route if route_id not found.
    Returns full Item dict or None.

  get_key(provider)
    Fetches provider key record from v3r-keys by key_id (provider name).
    Returns full Item dict or None.

  reset_if_new_day(provider, key_record, today)
    Lightweight daily reset check. If last_reset != today, sets
    requests_today to 0 and updates last_reset. EventBridge midnight
    reset is primary mechanism — this is a secondary safety reset.

  increment_usage(provider)
    Increments requests_today counter after every successful call.
    Uses UPDATE expression: requests_today = requests_today + 1.

  call_openai_compat(key_record, system_prompt, user_prompt, max_tokens,
                     temperature)
    OpenAI-compatible HTTP adapter. Handles Groq, Cerebras, Mistral,
    Fireworks. Uses http.client.HTTPSConnection (FP-002 compliant).
    base_url stored as hostname only — no https:// prefix.
    Sends POST to /v1/chat/completions with Bearer auth.
    Returns choices[0].message.content as plain text.

  route_llm(route_id, system_prompt, user_prompt, max_tokens=1024,
            temperature=0.3)
    Main entry point for all handlers.
    1. Resolves route_id via v3r-routing table.
    2. Iterates provider order (primary, fallback_1, fallback_2, fallback_3).
    3. Skips providers with status != 'active'.
    4. Resets daily counter if new day.
    5. Skips providers at or above daily_limit (marks as 'exhausted').
    6. Executes call on first available provider.
    7. Increments usage counter on success.
    8. Returns normalized text or error string if all exhausted.
