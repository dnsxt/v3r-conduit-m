Create public repo: v3r-nexus-mirror (GitHub → New repository → Public).
In repo, create .github/workflows/conduit-m-sync.yml:
name: CONDUIT-M Drive Mirror
on:
  schedule:
    - cron: '*/30 * * * *'
  workflow_dispatch:
jobs:
  mirror:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install rclone
        run: sudo apt update && sudo apt install -y rclone
      - name: Configure rclone (Drive readonly)
        run: |
          mkdir -p \~/.config/rclone
          cat > \~/.config/rclone/rclone.conf << EOF
          [v3rdrive]
          type = drive
          scope = drive.readonly
          token = ${{ secrets.DRIVE_TOKEN }}
          EOF
      - name: Mirror key folders
        run: |
          rclone sync v3rdrive:v3rframeworks/current . --delete-excluded --fast-list
          rclone sync v3rdrive:core/reports . --delete-excluded --fast-list
          rclone sync v3rdrive:conduit/logs . --delete-excluded --fast-list
          rclone sync v3rdrive:frameworks/current . --delete-excluded --fast-list
      - name: Commit & push (full history)
        run: |
          git config user.name "V3R CONDUIT-M"
          git config user.email "v3r@automated"
          git add .
          git commit -m "CONDUIT-M mirror $(date -u +'%Y-%m-%d %H:%M:%S UTC')" || echo "No changes"
          git push
Add secret: Repo Settings → Secrets → DRIVE_TOKEN (value = full [v3rdrive] section from your local rclone config show).
Push workflow file → first run triggers automatically.
