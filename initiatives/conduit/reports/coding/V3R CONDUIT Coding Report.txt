ISSUES ENCOUNTERED AND RESOLVED
================================================================================

ISSUE 1: rclone --no-delete flag does not exist
  Error: "rclone.exe : Error: unknown flag: --no-delete"
  Root cause: --no-delete is a rclone sync flag, not a copy flag. rclone copy
  never deletes destination files by default — the flag is both unsupported
  and unnecessary.
  Resolution: Removed all --no-delete flags from conduit_sync.ps1 using
  Notepad++ Find and Replace.
  FP added: FP-019

ISSUE 2: rclone OAuth token expired mid-sync
  Error: "couldn't fetch token: dial tcp: lookup oauth2.googleapis.com:
  no such host — Fatal error: couldn't find root directory ID"
  Root cause: rclone's cached OAuth token expired during sync run. Network
  connectivity to oauth2.googleapis.com was lost or token cache was stale.
  Resolution: Ran .\rclone.exe config reconnect gdrive: to re-authenticate.
  Browser opened, signed in with dnsxt.mbx@gmail.com, token refreshed.

ISSUE 3: ecr-build directory cluttered with session working .json files
  Cause: All Phase 7 and Phase 8 payload, response, and test .json files
  remained in ecr-build after those sessions. rclone pushed all of them to
  Drive on first sync.
  Resolution: Operator deleted all session working files matching:
  p7_*.json, phase8_*.json, k[1-7].json, kx[1-3].json, ri[1-6].json,
  graph_config.json, graph_table.json, groq_test.json, ideagen_resp.json,
  ideagen_test.json, logging.json, orch_env.json
  Retained: env_vars.json, orchestrator_config.json

ISSUE 4: Apps Script Drive.Files.copy conversion failure
  Error: "The requested conversion is not supported" on all .py, .json,
  Dockerfile files.
  Root cause: rclone uploads files with non-text/plain MIME types.
  drive.files.copy {convert:true} only works for text/plain.
  Resolution: Replaced with DocumentApp.create() approach that reads raw
  file content and creates Google Doc from scratch. Works for all MIME types.

NEW FORBIDDEN PATTERN ADDED THIS SESSION

FP-019: Using --no-delete flag with rclone copy command
  Error: "rclone.exe : Error: unknown flag: --no-delete"
  Root cause: --no-delete is a flag for rclone sync, not rclone copy.
  rclone copy never deletes destination files — the flag is unnecessary
  and unsupported on the copy subcommand.
  Fix: Remove --no-delete from all rclone copy commands. Use rclone copy
  (not rclone sync) and rely on its default non-destructive behavior.
  Reinforces FP-018: Never use rclone sync — always use rclone copy.
  Discovered: March 15, 2026 — CONDUIT CD-07 initial sync attempt.
