  Each provider now specifies its own endpoint path in DynamoDB.

  added without a path field — but path field is now required by FP-019.

FP-019: Hardcoding a single API path in lambda_router for all providers

  Error produced:
    No explicit error — silent failover. Groq fails on every call.
    Cerebras catches as fallback_1. requests_today for Groq remains 0.
    Root cause invisible without inspecting v3r-keys counters.

  Root cause:
    OpenAI-compatible providers do not all share the same API endpoint path.
    Groq: /openai/v1/chat/completions
    Cerebras: /v1/chat/completions
    Mistral: /v1/chat/completions
    Fireworks: /inference/v1/chat/completions
    Hardcoding /v1/chat/completions causes Groq to return an error
    (likely 404 or auth rejection) on every call.

  Fix:
    Store path field in v3r-keys per provider.
    lambda_router reads: path = key_record.get('path', '/v1/chat/completions')
    Every provider record in v3r-keys must include a path field.
    Never assume path for a new provider — verify against provider docs first.

  Detection method:
    Check requests_today in v3r-keys for each provider.
    A provider showing 0 requests when it is primary on active routes
    indicates silent failure. Investigate immediately.

  Discovered: March 18, 2026.
