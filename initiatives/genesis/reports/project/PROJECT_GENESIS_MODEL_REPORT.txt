------------------------------------------------------------------------
DATA FLOW (SINGLE CYCLE)
------------------------------------------------------------------------

1. Grok generates PNG containing compressed JSON payload
2. PNG appears in Grok conversation on Android
3. phone_tap.py detects image (color/ADB trigger), initiates download
4. PNG lands in watch folder
5. receiver.py fires, passes PNG path to decoder.py
6. decoder.py:
   a. Opens PNG in strict 1-bit mode
   b. Validates MAGIC bytes ("GENESIS")
   c. Parses header (version, frame_id, total_frames, payload_len, encoding)
   d. Extracts payload bytes
   e. CRC check
   f. Decompresses (ZLIB)
   g. JSON parse
   h. Returns structured payload object or typed error
7. validator.py confirms payload object matches spec
8. executor.py receives payload object:
   a. Checks cmd field against registered handlers
   b. Dispatches to handler with args
   c. Handler executes, captures output
   d. Returns result object
9. logger.py appends full record
10. return_formatter.py builds return JSON
11. transmitter.py ADB-injects return JSON into Grok conversation
12. Grok reads return, cycle complete

------------------------------------------------------------------------
COMMAND VOCABULARY (MVP — PART 1)
------------------------------------------------------------------------

SHELL
  args: { "cmd": "<shell command string>" }
  executes via subprocess, returns stdout + stderr
  timeout enforced, process killed on breach

PYTHON
  args: { "script": "<python source string>" }
  executes via subprocess with python interpreter
  stdout/stderr captured and returned
  no import restrictions in Phase 1 (full access design)

FILE_READ
  args: { "path": "<absolute or relative path>" }
  returns file contents as string

FILE_WRITE
  args: { "path": "<path>", "content": "<string>", "mode": "w|a" }
  writes content to file, returns success/fail

FILE_LIST
  args: { "path": "<directory path>" }
  returns directory listing with file sizes and timestamps

STATUS
  args: {}
  returns full system state snapshot:
    hostname, OS, Python version, CWD, disk usage, memory,
    executor status, handler list, last 3 log entries

HALT
  args: { "reason": "<optional string>" }
  immediately suspends executor loop
  clears pending queue
  returns HALTED status with reason
  system resumes only on manual restart or scheduled resume

------------------------------------------------------------------------
PHASE 1A — DECODER.PY
------------------------------------------------------------------------

PRIORITY: HIGHEST. Nothing works without this.

RULES:
  - NEVER use .convert('1') — open raw, check mode, reject if not '1'
  - NEVER use int(bit_string, 2) for large payloads — streaming byte rebuild
  - ALL validation failures return typed error objects, never raise uncaught

PSEUDOCODE:

def decode(png_path):
    im = Image.open(png_path)
    if im.mode != '1':
        return Error("NOT_1BIT", non-recoverable)

    pixels = np.array(im, dtype=np.uint8).flatten()
    pixels = (pixels > 0).astype(np.uint8)

    if len(pixels) != 262144:
        return Error("WRONG_PIXEL_COUNT", non-recoverable)

    byte_array = streaming_rebuild(pixels)     # 8 bits per byte, no int() overflow

    magic = byte_array[0:8].decode('ascii', errors='replace')
    if magic != 'GENESIS':
        return Error("MAGIC_FAIL", non-recoverable)

    header = parse_header(byte_array[8:40])    # 256 bits = 32 bytes
    payload_bytes = byte_array[40 : 40 + header.payload_len]

    crc_stored = read_crc_block(byte_array)
    crc_computed = crc32(payload_bytes)
    if crc_stored != crc_computed:
        return Error("CRC_FAIL", non-recoverable)

    if header.encoding == 1:
        payload_bytes = zlib.decompress(payload_bytes)

    payload_obj = json.loads(payload_bytes.decode('utf-8'))
    return payload_obj

------------------------------------------------------------------------
PHASE 1B — EXECUTOR.PY
------------------------------------------------------------------------

DESIGN PRINCIPLE: Router only. No logic in executor core.
All logic lives in handlers. Executor finds the handler, calls it, returns.

PSEUDOCODE:

HANDLER_REGISTRY = {
    "SHELL":      handlers.shell.run,
    "PYTHON":     handlers.python_run.run,
    "FILE_READ":  handlers.file_ops.read,
    "FILE_WRITE": handlers.file_ops.write,
    "FILE_LIST":  handlers.file_ops.list_dir,
    "STATUS":     handlers.status.run,
    "HALT":       handlers.halt.run,
}

def execute(payload_obj):
    cmd = payload_obj.get('cmd', '').upper()
    if cmd not in HANDLER_REGISTRY:
        return result(status="FAIL", error={"type": "UNKNOWN_CMD"})

    handler = HANDLER_REGISTRY[cmd]
    timeout = payload_obj.get('flags', {}).get('timeout', 30)
    args = payload_obj.get('args', {})

    try:
        output = run_with_timeout(handler, args, timeout)
        return result(status="OK", output=output)
    except TimeoutError:
        return result(status="TIMEOUT")
    except Exception as e:
        return result(status="FAIL", error={"type": type(e).__name__,
                                             "message": str(e)})

PHASE 1D — PHONE_TAP.PY
------------------------------------------------------------------------

PURPOSE: Automate the download sequence on Android when Grok responds with PNG.

MECHANISM OPTIONS (in order of preference):
  1. File system watcher on Android download folder via ADB (cleanest)
     -- adb shell inotifywait on Downloads folder
     -- when new .png file appears: pipeline triggers immediately
     -- no screen interaction needed if Grok auto-downloads

  2. Color detection on screen via ADB screencap + analysis
     -- adb exec-out screencap -p > screen.png
     -- scan for pixel signature of Grok PNG response card
     -- issue adb shell input tap <x> <y> to trigger download
     -- wait for file in Downloads via inotifywait

  3. uiautomator dump (most robust against layout changes)
     -- adb shell uiautomator dump /sdcard/ui.xml
     -- parse XML for download button or image element
     -- tap by element bounds, not hardcoded coordinates

RECOMMENDED FOR PART 1: Option 2 (color detection) for speed, with Option 3
as fallback. Option 1 only if Grok auto-downloads without tap required.

KEY ADB COMMANDS USED:
  adb exec-out screencap -p > screen.png
  adb shell input tap <x> <y>
  adb shell input swipe <x1> <y1> <x2> <y2> <ms>
  adb pull /sdcard/Download/<filename> <local_path>
  adb shell uiautomator dump /sdcard/ui.xml
  adb pull /sdcard/ui.xml <local_path>

------------------------------------------------------------------------
PHASE 1E — TRANSMITTER.PY
------------------------------------------------------------------------

PURPOSE: Get return JSON back into Grok's conversation.

MECHANISM:
  1. Format return JSON as compact string
  2. adb shell input text '<json_string>'
     NOTE: Special characters must be escaped for ADB text input.
     Single quotes, backslashes, spaces all need handling.
  3. adb shell input keyevent 66   (ENTER / send)

ALTERNATIVE if text injection is unreliable:
  1. Write return to a .txt file
  2. Use ADB to trigger a share/open action that pastes into Grok
  3. Or: display as QR code and have Grok read it (complex, avoid)

RECOMMENDED: Direct adb input text with proper escaping. If character limits
are hit (ADB text injection has practical limits ~500 chars), split into
multiple injections and prefix with [1/N], [2/N] for Grok to reassemble.

PHASE 1F — PIPELINE.PY
------------------------------------------------------------------------

PSEUDOCODE (full orchestration loop):

def main_loop():
    watcher = FileWatcher(config.WATCH_FOLDER)
    while True:
        png_path = watcher.wait_for_new_png()
        logger.log("PNG_RECEIVED", png_path)

        payload = decoder.decode(png_path)
        if isinstance(payload, Error):
            transmitter.send(format_error_return(payload))
            continue

        validation = validator.validate(payload)
        if not validation.ok:
            transmitter.send(format_error_return(validation.error))
            continue

        result = executor.execute(payload)
        logger.log("EXECUTED", payload['id'], result['status'])

        return_json = return_formatter.format(result)
        transmitter.send(return_json)

        if executor.HALT_FLAG:
            logger.log("HALTED")
            break
