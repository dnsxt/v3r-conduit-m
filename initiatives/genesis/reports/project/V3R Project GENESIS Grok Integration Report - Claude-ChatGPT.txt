II. IMAGE DATA SPECIFICATION (CANONICAL FORMAT)
1. Base Format (Unchanged Core)
Resolution: 512 × 512 pixels
Bit Depth: 1-bit (strict binary)
Encoding:
Black pixel = 0
White pixel = 1
Total Bits: 262,144 bits
Total Bytes: 32,768 bytes
Scan Order: Row-major (left → right, top → bottom)
2. Frame Segmentation (NEW – REQUIRED)
The image is no longer treated as a flat payload. It is now a structured frame:
Plain text
Rows 0–3     → SYNC + MAGIC
Rows 4–7     → HEADER
Rows 8–503   → PAYLOAD
Rows 504–507 → CRC32 (repeated)
Rows 508–511 → FOOTER / CONTROL
3. SYNC + MAGIC (NEW)
First 64 bits:
Plain text
"GENESIS"
Purpose:
Detect alignment errors instantly
Prevent decoding garbage data
4. HEADER STRUCTURE (NEW)
Fixed-size binary header (~256 bits):
Plain text
MAGIC        (64 bits)
VERSION      (8 bits)
FLAGS        (8 bits)
FRAME_ID     (32 bits)
TOTAL_FRAMES (32 bits)
PAYLOAD_LEN  (32 bits)
ENCODING     (8 bits)   (0=UTF8, 1=ZLIB)
RESERVED     (72 bits)
