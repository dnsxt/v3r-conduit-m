SECTION 1: PROJECT OVERVIEW AND VISION
This project is the construction of a cloud-native, serverless Retrieval-Augmented Generation (RAG) system. The immediate purpose of this system is to give AI assistants like Grok persistent memory and a personal knowledge base that survives across individual conversation sessions.
The core problem being solved is well-documented: AI systems like Grok, Claude, and ChatGPT have no built-in memory beyond the current conversation context window. Users must repeatedly provide the same background information, preferences, or custom knowledge in each new session. This RAG system solves that by allowing document/text ingestion into a vector database, with retrieval to augment queries for personalized, memory-aware responses.
The system is designed to be 100% free-tier compliant, using AWS free tier for storage and compute, Groq for fast inference, and local Python/PowerShell for development/testing. No credit card required services beyond the existing AWS free-tier account.
This is Phase 1 of an 8-phase roadmap to build an automated product generation platform, starting with basic RAG memory to bootstrap AI capabilities iteratively. The vision is to enable AIs to maintain user-specific knowledge (e.g., preferences, history, custom data) for more useful, context-aware interactions.
Key principles:
Bootstrap minimal viable RAG first to enhance AI reasoning.
Scale iteratively using the improved system to tackle larger tasks.
Strict free-tier adherence to avoid any billing risk.
Human-in-the-loop for verification at every stage.

SECTION 2: LONG-TERM SYSTEM ARCHITECTURE VISION
The long-term vision is an 8-phase evolution from a simple RAG memory system to a full Science and Engineering Intelligence Platform capable of automated product generation from natural language ideas.
PHASE 1: Basic RAG Memory System (Current MVP)
Text ingestion via Lambda (rag-ingest) with local embeddings (sentence-transformers all-MiniLM-L6-v2).
Vector storage in DynamoDB (rag_memory table with vector_id, embedding, text).
Retrieval via cosine similarity scan in Lambda (rag-retrieve).
Augmented generation via Groq API in Lambda (rag-chat).
Frontend: Optional Hugging Face Space or direct API calls for testing.
Persistence: Conversation history in rag_conversations table.
PHASE 2: Enhanced RAG with Conversation Persistence
Automatic conversation history ingestion/retrieval.
Session-based memory grouping.
Basic query rephrasing for better retrieval.
PHASE 3: Multi-Modal RAG
Image/PDF ingestion (OCR/extract text + embed).
Audio/video transcript + embed.
PHASE 4: Graph Neural Network Integration
Use GNN (e.g., PyG via Groq) for knowledge graph from RAG data.
Relation extraction and entity linking.
PHASE 5: Multi-Agent System
Agentic workflows (tool use, planning) integrated with RAG.
Self-improving agents using memory.
PHASE 6: Advanced Multi-Modal Capabilities
Image/CAD analysis via vision models.
Sensor data anomaly detection.
PHASE 7: Science and Engineering Intelligence Platform
Domain-fine-tuned models.
Automated literature review/synthesis.
Workflow automation for design/QC.
PHASE 8: Automated Product Generation
Input: Natural language ideas.
Output: Full products (specs, CAD, code) via integrated stack.
Human verification loops.
The architecture is fully serverless, scalable, and free-tier optimized, with AWS (Lambda, DynamoDB, ECR), Groq for inference, local Python for dev.
