### Bug #1 — get_history Was Using Wrong DynamoDB Method

The rag-memory table has a COMPOSITE KEY: partition key vector_id (S) AND sort key timestamp (N).

The original get_history() called memory_table.get_item(Key={"vector_id": session_id}). DynamoDB get_item requires ALL keys when a sort key exists. Since timestamp was not provided, get_item always returned empty silently. History never loaded. The chat had no memory of prior turns even when turns were being saved.

The fix (implemented in lambda_chat.py today):

```python
def get_history(session_id):
    try:
        resp = memory_table.query(
            KeyConditionExpression="vector_id = :sid",
            ExpressionAttributeValues={":sid": session_id},
            ScanIndexForward=True
        )
        items = resp.get("Items", [])
        if items:
            return items[-1].get("turns", [])
        return []
    except Exception:
        return []
```

The fix for save_history() to include the sort key timestamp:

```python
def save_history(session_id, turns):
    try:
        memory_table.put_item(Item={
            "vector_id": session_id,
            "timestamp": int(datetime.now(timezone.utc).timestamp()),
            "turns": turns,
            "updated": datetime.now(timezone.utc).isoformat()
        })
    except Exception:
        pass
```
