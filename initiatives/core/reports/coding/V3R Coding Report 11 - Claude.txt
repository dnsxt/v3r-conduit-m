FP-015: Omitting --cli-binary-format raw-in-base64-out from aws lambda invoke
        with --payload file:// commands

  Error produced:
    aws: [ERROR]: Invalid base64: {"action": "add_node", ...}

  Root cause:
    AWS CLI v2 expects base64-encoded payloads by default. When using
    --payload file:// with a plain JSON file, the CLI attempts to base64
    decode the JSON content and fails. The flag --cli-binary-format
    raw-in-base64-out tells the CLI to treat the file as raw text.

  Fix:
    Add --cli-binary-format raw-in-base64-out to every aws lambda invoke
    command that uses --payload file://.

  Correct pattern:
    aws lambda invoke --function-name [name] --region us-east-1 --cli-binary-format raw-in-base64-out --payload file://C:\absolute\path\payload.json C:\absolute\path\response.json

  Discovered: March 14, 2026 — Phase 7 Step 3 graph node operations.
