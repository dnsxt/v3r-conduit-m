SECTION 7 — LESSONS LEARNED THIS SESSION
================================================================================

RCLONE TOKEN EXPIRY:
  rclone OAuth tokens expire. If sync fails with "couldn't fetch token" or
  "no such host" errors, run:
  cd C:\Users\Source-1\Desktop\rclone
  .\rclone.exe config reconnect gdrive:
  Re-authenticate in browser. Token refreshed.

APPS SCRIPT MIME TYPE LIMITATION:
  Drive.Files.copy with {convert:true} only converts text/plain files.
  rclone uploads files with language-specific or binary MIME types.
  Fix: Use DocumentApp.create() to read file content and create Doc from
  scratch. This works for all MIME types regardless of source.

ECRBUILD CLEANUP:
  Session working .json files accumulate in ecr-build over time.
  Clean these before running conduit_sync to prevent Drive pollution.
  Safe to delete: p7_*.json, phase8_*.json, k*.json, kx*.json, ri*.json,
  test payloads and response files.
  Keep: env_vars.json, orchestrator_config.json.

RCLONE COPY VS SYNC:
  Always use rclone copy. Never rclone sync.
  rclone copy never deletes destination files.
  rclone sync deletes files on destination not present in source.
  --no-delete flag does not exist on rclone copy — it is redundant anyway.
