### What V3R Is
V3R is not a chatbot. Not a tool. It is the foundational architecture for an AI system that accepts raw ideas and outputs fully engineered products. ### Why V3R Exists
V3R solves the single most critical failure of all current AI tools: amnesia. Every standard AI session starts from zero — no history, no accumulated context, no engineering memory. V3R's persistent memory layer fixes this permanently. Knowledge ingested today is retrieved tomorrow. Conversations accumulate into a growing intelligence base. Engineering decisions are remembered across sessions. The system becomes more capable with every interaction. This is not a feature. It is the entire architectural thesis.

**ACTIVE BUG:** `get_history()` and `save_history()` functions are defined in the file (confirmed visually in photos) BUT the `rag-memory` table has BOTH a partition key `vector_id (S)` AND a sort key `timestamp (N)`. The current `get_history()` uses `get_item` which requires BOTH keys — it only provides `vector_id`, so it always returns empty history. This is why the chat returns "No response" — history never loads, context is always empty.

**REQUIRED FIX — replace get_history() at line 17:**
```python
def get_history(session_id):
    try:
        resp = memory_table.query(
            KeyConditionExpression="vector_id = :sid",
            ExpressionAttributeValues={":sid": session_id},
            ScanIndexForward=True
        )
        items = resp.get("Items", [])
        if items:
            return items[-1].get("turns", [])
        return []
    except Exception:
        return []
```

**REQUIRED FIX — replace save_history() at line 24:**
```python
def save_history(session_id, turns):
    try:
        memory_table.put_item(Item={
            "vector_id": session_id,
            "timestamp": int(datetime.now(timezone.utc).timestamp()),
            "turns": turns,
            "updated": datetime.now(timezone.utc).isoformat()
        })
    except Exception:
        pass
```
