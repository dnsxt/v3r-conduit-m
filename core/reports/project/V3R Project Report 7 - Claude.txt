SECTION 1: SESSION SUMMARY
This session completed Phase 1 of the V3R roadmap. All three core Lambda functions are now fully operational. The RAG pipeline is end-to-end functional for the first time.
SECTION 2: WHAT WAS BUILT THIS SESSION
Three Lambda functions are now live and working together:
rag-ingest-ultra-test — Accepts text input, chunks it, embeds it locally using all-MiniLM-L6-v2 via AutoTokenizer/AutoModel from transformers, and writes chunks to DynamoDB rag-chunks table with chunk_id, text, embedding, source, and timestamp fields.
rag-retrieve — Accepts a query, embeds it using the same local model, scans rag-chunks, runs cosine similarity against all stored embeddings, and returns the top-k most relevant chunks. Skips records with no embedding field.
rag-chat — Accepts a query, invokes rag-retrieve to get context chunks, sends the query and context to Groq API using llama-3.1-8b-instant, and returns a memory-augmented generated answer with source attribution.

SECTION 4: KEY TECHNICAL DECISIONS MADE THIS SESSION
SentenceTransformer() was abandoned in favor of AutoTokenizer/AutoModel from transformers with manual mean pooling. This bypasses HuggingFace hub validation entirely and loads the model from the baked-in local path /var/task/models/all-MiniLM-L6-v2 without any network calls.
urllib.request was replaced with http.client.HTTPSConnection for Groq API calls for more precise header control.
llama3-8b-8192 was replaced with llama-3.1-8b-instant after confirming the former was decommissioned by Groq.
Tag ml-offline-torch26 is the permanent production tag. Do not switch tags without explicit instruction.
All three handlers are baked into the same container image. CMD override in Lambda Image Configuration selects which handler runs per function.
