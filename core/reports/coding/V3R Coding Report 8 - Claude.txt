## SECTION 2 — lambda_chat.py REWRITES

### CONTEXT
lambda_chat.py was audited via photos taken at 2:51 PM. The file was 79 lines and already contained get_history and save_history functions. However the functions used `memory_table.get_item` which fails silently when the table has a composite key (vector_id + timestamp). History never loaded. Chat had no memory across turns.

### CHANGE 1 — get_history() REWRITE
**Status: SUCCEEDED (implemented, deployed)**

Old code (broken):
```python
def get_history(session_id):
    try:
        resp = memory_table.get_item(Key={"vector_id": session_id})
        return resp.get("Item", {}).get("turns", [])
    except Exception:
        return []
```

New code (correct):
```python
def get_history(session_id):
    try:
        resp = memory_table.query(
            KeyConditionExpression="vector_id = :sid",
            ExpressionAttributeValues={":sid": session_id},
            ScanIndexForward=True
        )
        items = resp.get("Items", [])
        if items:
            return items[-1].get("turns", [])
        return []
    except Exception:
        return []
```

**Why:** rag-memory table has composite key (vector_id PK + timestamp SK). get_item requires both keys. query only needs the partition key and returns all matching records. Fixed the silent failure.

### CHANGE 2 — save_history() REWRITE
**Status: SUCCEEDED (implemented, deployed)**

Old code (broken):
```python
def save_history(session_id, turns):
    try:
        memory_table.put_item(Item={"vector_id": session_id, "turns": turns, "updated": datetime.now(timezone.utc).isoformat()})
    except Exception:
        pass
```

New code (correct):
```python
def save_history(session_id, turns):
    try:
        memory_table.put_item(Item={
            "vector_id": session_id,
            "timestamp": int(datetime.now(timezone.utc).timestamp()),
            "turns": turns,
            "updated": datetime.now(timezone.utc).isoformat()
        })
    except Exception:
        pass
```

**Why:** put_item to a composite key table requires both PK and SK. Adding `timestamp` as integer unix epoch satisfies the sort key requirement.
